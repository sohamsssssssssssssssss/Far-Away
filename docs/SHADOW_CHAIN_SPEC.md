# Shadow-Journal chain spec — for the frontend session

Your `ShadowJournal.tsx` in-browser verify is self-consistent on its own sample,
but it does **not** match the real `disastermind/ml/shadow.py` algorithm — so it
would show "chain broken" on genuine shadow data. Align to the spec below and the
badge becomes real (and future-proof).

## The exact algorithm (from `disastermind/ml/shadow.py::_chain_hash`)

```
genesis  = "shadow-genesis"                 # NOT "GENESIS"
for each record in order:
    expected = SHA256_hex( prevHash + "|" + canonicalJSON(record.payload) )
    assert record.hash === expected
    prevHash = record.hash
```

- **Literal `"|"` separator** between prevHash and the body.
- **`canonicalJSON`** = JSON with **recursively sorted keys** and **compact
  separators** `","` and `":"` (no spaces). Equivalent to Python
  `json.dumps(payload, sort_keys=True, separators=(",", ":"))`.
  Plain `JSON.stringify(payload)` will **not** work (it doesn't sort keys).
- Hash is over `record.payload` only (the `hash`/`kind` fields are excluded).
- Output is lowercase hex.

## Minimal JS (crypto.subtle)

```js
function canonical(v) {
  if (Array.isArray(v)) return "[" + v.map(canonical).join(",") + "]";
  if (v && typeof v === "object")
    return "{" + Object.keys(v).sort()
      .map(k => JSON.stringify(k) + ":" + canonical(v[k])).join(",") + "}";
  return JSON.stringify(v);            // strings/numbers/bool/null
}
async function sha256Hex(s) {
  const b = await crypto.subtle.digest("SHA-256", new TextEncoder().encode(s));
  return [...new Uint8Array(b)].map(x => x.toString(16).padStart(2, "0")).join("");
}
async function verifyChain(records, genesis = "shadow-genesis") {
  let prev = genesis;
  for (const r of records) {
    const expected = await sha256Hex(prev + "|" + canonical(r.payload));
    if (r.hash !== expected) return false;
    prev = r.hash;
  }
  return true;
}
```

(note the wrapper key is **`journal`**, not `records`). Read `.journal` for the
records and `.scorecard` for season metrics.
